This page lists all available flags and their usage across commands.

## Global flags

These are flags that can be used with any command.

### --config

Server config file (default "resonate.yml")

### --log-level

Log level, can be one of:

- debug
- info (default)
- warn
- error

### --password

Basic auth password

### --server

Server address used by the client (default "http://127.0.0.1:8001").

### --username

Basic auth username

## Command flags

These are flags that are specific to a command.

### --cron

Schedule cron expression.

Used in:

- [schedules create](/operate/cli-reference/commands/schedule#create)

### --description

Schedule description.

Used in:

- [schedules create](/operate/cli-reference/commands/schedule#create)

### --idempotency-key

An optional idempotency key to ensure idempotency in promise operations.

Used in:

- [promises create](/operate/cli-reference/commands/promise#create)
- [promises resolve](/operate/cli-reference/commands/promise#resolve)
- [promises reject](/operate/cli-reference/commands/promise#reject)
- [promises cancel](/operate/cli-reference/commands/promise#cancel)

### --strict

Enforces strict mode in promise operations.
Default is true.

Used in:

- [promises create](/operate/cli-reference/commands/promise#create)
- [promises resolve](/operate/cli-reference/commands/promise#resolve)
- [promises reject](/operate/cli-reference/commands/promise#reject)
- [promises cancel](/operate/cli-reference/commands/promise#cancel)

### --timeout

Specifies the timeout duration for promises

Used in:

- [promises create](/operate/cli-reference/commands/promise#create)

### --promise-timeout

Specifies the timeout duration for promises in Schedules.

Used in:

- [schedules create](/operate/cli-reference/commands/schedule#create)

### --data

Specifies the data for promises.

Used in:

- [promises create](/operate/cli-reference/commands/promise#create)
- [promises resolve](/operate/cli-reference/commands/promise#resolve)
- [promises reject](/operate/cli-reference/commands/promise#reject)

### --header

Specifies headers for promise requests.

Used in:

- [promises create](/operate/cli-reference/commands/promise#create)
- [promises resolve](/operate/cli-reference/commands/promise#resolve)
- [promises reject](/operate/cli-reference/commands/promise#reject)

### --tag

Assigns tags to promises or schedules.

Used in:

- [promises create](/operate/cli-reference/commands/promise#create)

### --promise-tag

Promise tags. Default is `[]`.

Used in:

- [schedules create](/operate/cli-reference/commands/schedule#create)

### --promise-data

Promise param data

Used in:

- [schedules create](/operate/cli-reference/commands/schedule#create)

### --promise-header

Promise param header.
Default is `[]`.

Used in:

- [schedules create](/operate/cli-reference/commands/schedule#create)

### --promise-id

Templated schedule id, can include `{{.timestamp}}`.

Used in:

- [schedules create](/operate/cli-reference/commands/schedule#create)

### --state

Filters promises by state.

Used in:

- [promises search](/operate/cli-reference/commands/promise#search)

### --limit

Limits the number of results in searches.

Used in:

- [promises search](/operate/cli-reference/commands/promise#search)
- [schedules search](/operate/cli-reference/commands/schedule#search)

### --cursor

Pagination cursor for search results.

Used in:

- [promises search](/operate/cli-reference/commands/promise#search)
- [schedules search](/operate/cli-reference/commands/schedule#search)

### --output

Specifies the output format.

Used in:

- [promises get](/operate/cli-reference/commands/promise#get)
- [schedules get](/operate/cli-reference/commands/schedule#get)

Specifies the size of the submission queue buffered channel for the API.

- [serve](/operate/cli-reference/commands/serve)

- [serve](/operate/cli-reference/commands/serve)

- [serve](/operate/cli-reference/commands/serve)
